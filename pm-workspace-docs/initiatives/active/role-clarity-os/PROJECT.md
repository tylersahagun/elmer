# Role Clarity & Operating System

## What This Is

A systematic analysis project to identify the gap between stated role expectations, actual work patterns, and ideal focus areas for Tyler's PM position at AskElephant. The project will produce a comprehensive role definition document, personal operating system for decision-making, and actionable plan to align work with expectations while establishing sustainable boundaries.

## Core Value

Create clarity between what Tyler thinks his job is, what leadership expects, and what he's actually doing - then establish a sustainable operating model that focuses on his superpower (cross-functional coordination) while eliminating wheel-spinning on low-value work.

## Requirements

### Validated

- ✓ Comprehensive PM workspace infrastructure with 30+ commands, 20+ skills, and multiple agents — existing
- ✓ Integration pipelines for Linear, Notion, Slack, GitHub, PostHog data collection — existing
- ✓ Initiative lifecycle management and roadmap analysis capabilities — existing
- ✓ Signal synthesis from multiple communication channels — existing
- ✓ Activity reporting and digest generation systems — existing

### Active

- [ ] **Expectation Analysis**: Extract stated expectations from 1-on-1 conversations (Sam, Brian, Jamis), Slack channels, and direct assignments
- [ ] **Work Pattern Analysis**: Analyze last 2-3 months of actual work via PM workspace commits, Linear assignments, GitHub activity, and time allocation
- [ ] **Gap Identification**: Map disconnect between stated expectations, Tyler's interpretation, and actual work execution
- [ ] **Superpower Definition**: Identify core strength (cross-functional coordination) and how it's being under-utilized or over-extended
- [ ] **Priority Framework**: Create decision rules for what to say yes/no to, based on role boundaries and leverage
- [ ] **Scope Simplification**: Identify PM workspace complexity that's out of scope and recommend simplification
- [ ] **Communication Plan**: Draft language for setting boundaries and renegotiating expectations with leadership
- [ ] **Weekly Operating Rhythm**: Design sustainable weekly structure that prevents context switching and overpromising

### Out of Scope

- Building new PM workspace features during this analysis — focus is understanding role, not expanding tooling
- Analyzing work patterns beyond 2-3 months — recent data is most relevant to current reality
- Deep technical audits of elephant-ai codebase — PM workspace and initiatives only
- Career development planning beyond current role clarity — focus is performing current role well

## Context

Tyler has built an extraordinarily sophisticated PM workspace with comprehensive automation, but is experiencing all four alignment pain points:
1. **Constant context-switching**: Too many different types of work, can't go deep
2. **Overpromising timelines**: Says yes to everything, misses deadlines, disappoints people
3. **Working on wrong things**: Effort on tasks that don't matter or aren't valued
4. **Unclear accountability**: Doesn't know what he's actually responsible for delivering

The PM workspace reveals someone who thinks their job is to build systems, automate workflows, and create comprehensive analysis infrastructure. However, recent 1-on-1 conversations suggest leadership may expect something different. Tyler tends to overpromise and underdeliver, dropping "glass balls" (critical deliverables) while spending time on "rubber balls" (nice-to-have automation).

Success in 3 months means: clear weekly priorities, aligned expectations with Sam/leadership, sustainable pace, and simplified scope focused on high-leverage work.

## Constraints

- **Timeline**: Analysis covers last 2-3 months only (most relevant to current reality)
- **Data sources**: All available - 1-on-1 notes, Slack channels/DMs, Linear assignments, GitHub commits, PM workspace docs, initiative tracking
- **Deliverable scope**: Must produce three artifacts: role definition document, personal operating system, and action plan
- **Focus**: Cross-functional coordination is the superpower to lean into, not pure PM strategy or pure technical execution
