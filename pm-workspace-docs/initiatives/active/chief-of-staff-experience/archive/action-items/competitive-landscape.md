# Competitive Landscape - Action Items

Date: 2026-02-17

## Shared competitor set

- Revenue intelligence: Gong, Clari, Chorus (ZoomInfo), Avoma, Fireflies.ai, Fathom
- CRM AI hubs: Salesforce Agentforce, HubSpot Breeze, Salesloft, Outreach
- Agent automation: Relay.app, Lindy.ai, StackAI, Gumloop, Zapier, Make

## Relevance filter for Action Items

Most relevant competitors are those that convert insights into executable next steps with governance. We score highly for:

1. Action acceptance/editability
2. Scheduling and follow-up execution speed
3. Evidence traceability for each recommended action
4. Queue-level prioritization and ownership

## Tiering

- **Direct:** Salesforce Agentforce, HubSpot Breeze, Salesloft, Outreach
- **Adjacent:** Gong, Clari, Avoma, Fireflies.ai, Fathom
- **Execution platforms:** Relay.app, Lindy.ai, StackAI, Gumloop, Zapier, Make

## Feature matrix

| Capability           | AskElephant target state                           | Common competitor pattern                  | Opportunity                                 |
| -------------------- | -------------------------------------------------- | ------------------------------------------ | ------------------------------------------- |
| Unified action queue | Prioritized cards with confidence + source context | Actions split across CRM and meeting tools | Single queue with clear rationale           |
| Action controls      | Approve, edit, snooze, schedule, escalate          | Binary accept/ignore or manual export      | Granular controls with fast execution       |
| Scheduling assist    | Calendar-aware follow-up proposals                 | Suggestion without availability context    | End-to-end scheduling from action card      |
| Ownership + SLA      | Action owner and due horizon defaults              | Weak ownership semantics                   | Built-in accountability loop                |
| Feedback loop        | Outcome capture to improve models                  | Limited post-action learning               | Closed-loop learning from execution results |

## Patterns to adopt

- Compact, triageable action cards
- Explicit urgency and confidence indicators
- Fast state transitions (done, snoozed, delegated)

## Patterns to reject

- Hidden automation side effects
- Action generation without evidence
- Fragmented queues by source system

## Differentiation opportunities

1. Best-in-class evidence-backed action queue
2. Calendar-native scheduling and follow-up conversion
3. Cross-signal de-duplication and prioritization
4. Governance controls for safe automation at scale
