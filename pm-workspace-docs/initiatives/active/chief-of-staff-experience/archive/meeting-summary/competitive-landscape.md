# Competitive Landscape: Meeting Summary

**Initiative:** Meeting Summary (Chief of Staff)
**Date:** 2026-02-17
**Analysis scope:** Shared competitor set — focus on summary delivery, templating, editability
**Owner:** Tyler

---

## TL;DR

Meeting Summary competes for **first-class, editable recap artifacts**. Gong/Clari/Chorus deliver fixed-format post-meeting briefs. Fathom/Fireflies offer shareable summaries but no in-context template swap or section-level AI edit. **No competitor offers templatized, editable summary as default artifact** with save-as-template persistence.

**Primary differentiation:**

1. **First-class artifact** — Summary is primary meeting output, not buried in workflow
2. **Template by meeting type** — User selects format without workflow reconfiguration
3. **Section-level AI edit** — Rewrite sections in place; persist as template default
4. **Evidence-backed** — Source quotes and confidence (inherited from Chief of Staff)

## Initiative-Specific Relevance Filter

Meeting Summary focuses on:

- **Summary delivery** — Where and how recap reaches user (inbox, in-app, Slack)
- **Template/format control** — Meeting-type formats, customization
- **Editability** — In-context edit, AI rewrite, section-level changes
- **Persistence** — Save edits as template, reuse across meetings

---

## Patterns to Adopt

- **Immediate delivery** (Gong, Chorus) — Post-meeting brief as soon as possible
- **Meeting-type extraction** (Fireflies) — Pre-built formats per meeting type
- **Shareable clips** (Fathom) — Polished output for external sharing

---

## Patterns to Reject

- **Fixed format only** — Users must be able to choose and edit
- **Workflow-hidden output** — Summary must not require workflow navigation to access
- **Raw model output feel** — Polished artifact, not raw AI text
